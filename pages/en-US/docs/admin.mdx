# Deployment Guide

## Environment Variables

Before deploying, make sure to set the following environment variables directly in your deployment commands to configure the application:

```env
# Default language for the application
NEXT_PUBLIC_DEFAULT_LANGUAGE=<your_default_language zh-CN | en-US>

# Site URL and API endpoint
NEXT_PUBLIC_SITE_URL=<your_site_url>
NEXT_PUBLIC_API_URL=<your_api_url>

# Contact email for support or inquiries
NEXT_PUBLIC_EMAIL=<your_contact_email>

# Community and social media links
NEXT_PUBLIC_GITHUB_LINK=<your_github_link>

# Default user credentials (for initial setup or testing purposes)
NEXT_PUBLIC_DEFAULT_USER_EMAIL=<your_default_user_email>
NEXT_PUBLIC_DEFAULT_USER_PASSWORD=<your_default_user_password>
```

Refer to the [example environment file](https://raw.githubusercontent.com/perfect-panel/ppanel-admin-web/refs/heads/main/.env.development) for a structured reference.

## Deploy on Vercel

To quickly deploy on Vercel, click the button below:

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https%3A%2F%2Fgithub.com%2Fperfect-panel%2Fppanel-admin-web&env=NEXT_PUBLIC_API_URL&envDescription=Environment%20Variables%20Description&envLink=https%3A%2F%2Fraw.githubusercontent.com%2Fperfect-panel%2Fppanel-admin-web%2Frefs%2Fheads%2Fmain%2F.env.development&project-name=ppanel-admin-web&repository-name=ppanel-admin-web)

### Steps to Deploy on Vercel

1. Log in to your [Vercel account](https://vercel.com/) or create one if necessary.
2. Click the **Deploy with Vercel** button above.
3. In the Vercel dashboard, set the required environment variables, such as `NEXT_PUBLIC_API_URL`.
4. Click **Deploy** to initiate the deployment.
5. After deployment, view the live URL in your Vercel dashboard.

## Deploy with Docker

### Run the Docker Container

1. **Using the Latest Version**:

   ```sh
   docker run -d -p 3000:3000 \
     --env NEXT_PUBLIC_DEFAULT_LANGUAGE=en-US \
     --env NEXT_PUBLIC_SITE_URL=https://example.com \
     --env NEXT_PUBLIC_API_URL=https://api.example.com \
     --env NEXT_PUBLIC_EMAIL=contact@example.com \
     --env NEXT_PUBLIC_GITHUB_LINK=https://github.com/example/repository \
     --env NEXT_PUBLIC_DEFAULT_USER_EMAIL=user@example.com \
     --env NEXT_PUBLIC_DEFAULT_USER_PASSWORD=password123 \
     --name ppanel-admin-web ppanel/ppanel-admin-web:latest
   ```

2. **Using the Beta Version**:

   ```sh
   docker run -d -p 3000:3000 \
     --env NEXT_PUBLIC_DEFAULT_LANGUAGE=en-US \
     --env NEXT_PUBLIC_SITE_URL=https://example.com \
     --env NEXT_PUBLIC_API_URL=https://api.example.com \
     --env NEXT_PUBLIC_EMAIL=contact@example.com \
     --env NEXT_PUBLIC_GITHUB_LINK=https://github.com/example/repository \
     --env NEXT_PUBLIC_DEFAULT_USER_EMAIL=user@example.com \
     --env NEXT_PUBLIC_DEFAULT_USER_PASSWORD=password123 \
     --name ppanel-admin-web-beta ppanel/ppanel-admin-web:beta
   ```

### Run with Docker Compose

3. **Using Docker Compose (Latest Version)**:
   Create a `docker-compose.yml` file with this configuration:

   ```yaml
   version: '3'

   services:
     ppanel-admin-web:
       image: ppanel/ppanel-admin-web:latest
       ports:
         - '3000:3000'
       environment:
         NEXT_PUBLIC_DEFAULT_LANGUAGE: en-US
         NEXT_PUBLIC_SITE_URL: https://example.com
         NEXT_PUBLIC_API_URL: https://api.example.com
         NEXT_PUBLIC_EMAIL: contact@example.com
         NEXT_PUBLIC_GITHUB_LINK: https://github.com/example/repository
         NEXT_PUBLIC_DEFAULT_USER_EMAIL: user@example.com
         NEXT_PUBLIC_DEFAULT_USER_PASSWORD: password123
   ```

   Start the service:

   ```sh
   docker-compose up -d
   ```

4. **Using Docker Compose (Beta Version)**:
   Create a `docker-compose-beta.yml` file with this configuration:

   ```yaml
   version: '3'

   services:
     ppanel-admin-web-beta:
       image: ppanel/ppanel-admin-web:beta
       ports:
         - '3001:3000'
       environment:
         NEXT_PUBLIC_DEFAULT_LANGUAGE: en-US
         NEXT_PUBLIC_SITE_URL: https://example.com
         NEXT_PUBLIC_API_URL: https://api.example.com
         NEXT_PUBLIC_EMAIL: contact@example.com
         NEXT_PUBLIC_GITHUB_LINK: https://github.com/example/repository
         NEXT_PUBLIC_DEFAULT_USER_EMAIL: user@example.com
         NEXT_PUBLIC_DEFAULT_USER_PASSWORD: password123
   ```

   Start the beta service:

   ```sh
   docker-compose -f docker-compose-beta.yml up -d
   ```
