# Deployment Guide

## Environment Variables

Before deploying, make sure to set the following environment variables directly in your deployment commands to configure the application:

```bash
# Default language for the application
NEXT_PUBLIC_DEFAULT_LANGUAGE=<your_default_language zh-CN | en-US>

# Site URL and API endpoint
NEXT_PUBLIC_SITE_URL=<your_site_url>
NEXT_PUBLIC_API_URL=<your_api_url>

# Contact email for support or inquiries
NEXT_PUBLIC_EMAIL=<your_contact_email>

# Community and social media links
NEXT_PUBLIC_TELEGRAM_LINK=<your_telegram_link>
NEXT_PUBLIC_TWITTER_LINK=<your_twitter_link>
NEXT_PUBLIC_DISCORD_LINK=<your_discord_link>
NEXT_PUBLIC_INSTAGRAM_LINK=<your_instagram_link>
NEXT_PUBLIC_LINKEDIN_LINK=<your_linkedin_link>
NEXT_PUBLIC_FACEBOOK_LINK=<your_facebook_link>
NEXT_PUBLIC_GITHUB_LINK=<your_github_link>

# Default user credentials (for initial setup or testing purposes)
NEXT_PUBLIC_DEFAULT_USER_EMAIL=<your_default_user_email>
NEXT_PUBLIC_DEFAULT_USER_PASSWORD=<your_default_user_password>
```

Refer to the [example environment file](https://github.com/perfect-panel/ppanel-web/blob/main/apps/user/.env.template) for a structured reference.

### Vercel Deployment Steps

1. Log in to your [Vercel account](https://vercel.com/), or create one if you don't have an account.
2. Click the **Deploy on Vercel** button above to fork the repository.
3. In the Vercel console, select the application directory within the `ppanel-web` that you want to deploy, e.g., `apps/user`.
4. Set the required environment variables, such as `NEXT_PUBLIC_API_URL`.
5. Click **Deploy** to start the deployment.
6. Once the deployment is complete, you can view the live URL in the Vercel console.

## Deploy with Docker

## Install Docker

To install Docker, run the following command:

```sh
curl -fsSL https://get.docker.com | bash -s docker
```

### Run the Docker Container

   ```sh
   docker run -d -p 3000:3000 \
     --env NEXT_PUBLIC_DEFAULT_LANGUAGE=en-US \
     --env NEXT_PUBLIC_SITE_URL=https://example.com \
     --env NEXT_PUBLIC_API_URL=https://api.example.com \
     --env NEXT_PUBLIC_EMAIL=contact@example.com \
     --env NEXT_PUBLIC_TELEGRAM_LINK=https://t.me/example \
     --env NEXT_PUBLIC_TWITTER_LINK=https://twitter.com/example \
     --env NEXT_PUBLIC_DISCORD_LINK=https://discord.com/example \
     --env NEXT_PUBLIC_INSTAGRAM_LINK=https://instagram.com/example \
     --env NEXT_PUBLIC_LINKEDIN_LINK=https://linkedin.com/example \
     --env NEXT_PUBLIC_FACEBOOK_LINK=https://facebook.com/example \
     --env NEXT_PUBLIC_GITHUB_LINK=https://github.com/example/repository \
     --env NEXT_PUBLIC_DEFAULT_USER_EMAIL=user@example.com \
     --env NEXT_PUBLIC_DEFAULT_USER_PASSWORD=password123 \
     --name ppanel-user-web ppanel/ppanel-user-web:latest
   ```

### Run with Docker Compose

  Create a `docker-compose.yml` file with this configuration:

   ```yaml
   version: '3'

   services:
     ppanel-user-web-beta:
       image: ppanel/ppanel-user-web:beta
       ports:
         - '3001:3000'
       environment:
         NEXT_PUBLIC_DEFAULT_LANGUAGE: en-US
         NEXT_PUBLIC_SITE_URL: https://example.com
         NEXT_PUBLIC_API_URL: https://api.example.com
         NEXT_PUBLIC_EMAIL: contact@example.com
         NEXT_PUBLIC_TELEGRAM_LINK: https://t.me/example
         NEXT_PUBLIC_TWITTER_LINK: https://twitter.com/example
         NEXT_PUBLIC_DISCORD_LINK: https://discord.com/example
         NEXT_PUBLIC_INSTAGRAM_LINK: https://instagram.com/example
         NEXT_PUBLIC_LINKEDIN_LINK: https://linkedin.com/example
         NEXT_PUBLIC_FACEBOOK_LINK: https://facebook.com/example
         NEXT_PUBLIC_GITHUB_LINK: https://github.com/example/repository
         NEXT_PUBLIC_DEFAULT_USER_EMAIL: user@example.com
         NEXT_PUBLIC_DEFAULT_USER_PASSWORD: password123
   ```

   Start service:

   ```sh
   docker compose -f docker-compose.yml up -d
   ```
