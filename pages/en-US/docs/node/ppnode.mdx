# Deploy PPanel Node

The current PPanel Node repository is
[perfect-panel/ppanel-node](https://github.com/perfect-panel/ppanel-node). The
old `wyx2685/ppanel-node` script is no longer an official installation source.

## Install

Run the script from the official repository on the node server:

```bash
wget -N https://raw.githubusercontent.com/perfect-panel/ppanel-node/master/scripts/install.sh
bash install.sh
```

After installation, use the configuration generated by the node project to set
the panel address, node ID, and communication secret. Do not put panel admin
secrets or database credentials in a node configuration.

## Protocol negotiation

The node negotiates the transport with `Accept: application/protobuf` while
pulling its configuration. If the panel returns Protobuf, later user-list,
online-user, traffic, and status reports use Protobuf too; otherwise the node
continues to use JSON. No feature flag is required.

A panel can expose inbound protocols not supported by a particular node
version. Those protocols are skipped without preventing supported inbounds
from starting.

## Build from source

```bash
git clone https://github.com/perfect-panel/ppanel-node.git
cd ppanel-node
GOEXPERIMENT=jsonv2 go build -v -o node -trimpath -ldflags "-s -w -buildid="
```

Use that repository's README and release notes as the source of truth for
runtime options and configuration fields.
