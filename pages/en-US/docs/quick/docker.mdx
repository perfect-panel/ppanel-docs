# Quick start: Docker

Deploy the backend first, then build and deploy the admin and user frontends
independently. The former all-in-one example depended on stale frontend images
and deprecated configuration fields, so it is no longer recommended.

## 1. Start the backend

```bash
git clone https://github.com/perfect-panel/server.git ppanel-server
cd ppanel-server
mkdir -p etc
: > etc/ppanel.yaml
```

Use this minimal Compose file to start the backend, recommended PostgreSQL,
and Redis. Replace every example password first.

```yaml
services:
  ppanel:
    build: .
    restart: unless-stopped
    ports: ["8080:8080"]
    environment:
      PPANEL_DB: postgres://ppanel:change-me@db:5432/ppanel?sslmode=disable&TimeZone=Asia%2FShanghai
      PPANEL_REDIS: redis://:change-me@redis:6379/0
    volumes: ["./etc:/app/etc"]
    depends_on: [db, redis]
  db:
    image: postgres:16-alpine
    restart: unless-stopped
    environment:
      POSTGRES_DB: ppanel
      POSTGRES_USER: ppanel
      POSTGRES_PASSWORD: change-me
    volumes: ["postgres-data:/var/lib/postgresql/data"]
  redis:
    image: redis:7-alpine
    restart: unless-stopped
    command: ["redis-server", "--requirepass", "change-me"]
    volumes: ["redis-data:/data"]
volumes:
  postgres-data:
  redis-data:
```

```bash
docker compose up -d --build
docker compose logs -f ppanel
```

After the first start, set `JwtAuth.AccessSecret` and a strong administrator
password in `etc/ppanel.yaml`, then run `docker compose restart ppanel`. The
[full Docker guide](../server/docker) includes health checks and security
guidance for production.

## 2. Deploy the frontends

The frontend repository is now
[perfect-panel/frontend](https://github.com/perfect-panel/frontend). It
contains Vite-built `apps/admin` and `apps/user`, rather than the old
`ppanel-web` project.

Set this in each application's `.env.production`:

```dotenv
VITE_API_BASE_URL=https://api.example.com
```

Follow the [frontend separation deployment guide](https://github.com/perfect-panel/frontend/blob/main/docs/guide/separation/frontend.md)
to build the static files and host them with Nginx, object storage/CDN, or
Vercel. Ensure the backend CORS and frontend domains agree.
