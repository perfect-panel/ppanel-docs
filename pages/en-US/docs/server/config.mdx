# Server configuration

PPanel reads `etc/ppanel.yaml` by default. Use `--config` to select another
file:

```bash
ppanel run --config /etc/ppanel/ppanel.yaml
```

On first startup, an empty default configuration file is initialized from
`PPANEL_DB` and `PPANEL_REDIS`. Persist the generated file and set a stable,
high-entropy JWT secret before using the service in production.

## Minimal configuration

```yaml
Model: prod
Host: 0.0.0.0
Port: 8080
AppLocation: Asia/Shanghai

JwtAuth:
  AccessSecret: replace-with-a-long-random-secret
  AccessExpire: 604800

Database:
  Driver: postgres
  Addr: 127.0.0.1:5432
  Dbname: ppanel
  Username: ppanel
  Password: replace-with-a-database-password
  Config: sslmode=disable&TimeZone=Asia/Shanghai
  MaxIdleConns: 10
  MaxOpenConns: 10
  SlowThreshold: 1000

Redis:
  Host: 127.0.0.1:6379
  Pass: ""
  DB: 0

Administrator:
  Email: admin@example.com
  Password: replace-with-a-strong-initial-password
```

`Database` is the current setting and supports `postgres` and `mysql`.
PostgreSQL is recommended for new deployments. The legacy `MySQL` setting
remains compatible but is deprecated and should not be used for new deployments.

## First-start environment variables

These variables are only used to initialize an empty default configuration
file. After initialization, manage the YAML file instead.

| Variable | Example |
| --- | --- |
| `PPANEL_DB` | `postgres://ppanel:password@127.0.0.1:5432/ppanel?sslmode=disable&TimeZone=Asia%2FShanghai` |
| `PPANEL_REDIS` | `redis://:password@127.0.0.1:6379/0` |

MySQL/MariaDB remains supported. When using it, a driver DSN is accepted:

```text
ppanel:password@tcp(127.0.0.1:3306)/ppanel?charset=utf8mb4&parseTime=true&loc=Asia%2FShanghai
```

## Requirements

- Go is required only when building from source.
- PostgreSQL is recommended; MySQL/MariaDB remains supported. Redis is used
  for cache and queues.
- Keep the database, Redis, and backend port on trusted networks; terminate
  public HTTPS traffic at a reverse proxy.
