# Build and deploy the server from source

For a self-built binary, use the current
[server repository](https://github.com/perfect-panel/server). Do not use the
old fixed-version download URLs or migrated repository names from earlier
documentation.

## Prerequisites

- Go 1.25 or later
- PostgreSQL (recommended; MySQL/MariaDB remains supported)
- Redis

Create a PostgreSQL database and a least-privileged account:

```sql
CREATE USER ppanel WITH PASSWORD 'replace-this-password';
CREATE DATABASE ppanel OWNER ppanel;
```

## Build and install

```bash
git clone https://github.com/perfect-panel/server.git
cd server
go build -trimpath -o ppanel .

sudo install -Dm755 ppanel /usr/local/bin/ppanel
sudo install -d -m 0750 /etc/ppanel
sudo install -m 0640 etc/ppanel.yaml /etc/ppanel/ppanel.yaml
```

Edit `/etc/ppanel/ppanel.yaml` and configure PostgreSQL with `Database`, not
the deprecated `MySQL` field. See [Server configuration](./config) for a
complete example. MySQL/MariaDB connections and migration remain supported,
but PostgreSQL is recommended for new deployments.

## systemd

Create `/etc/systemd/system/ppanel.service`:

```ini
[Unit]
Description=PPanel server
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=ppanel
Group=ppanel
WorkingDirectory=/var/lib/ppanel
ExecStart=/usr/local/bin/ppanel run --config /etc/ppanel/ppanel.yaml
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

Create the runtime user and start the service:

```bash
sudo useradd --system --home /var/lib/ppanel --shell /usr/sbin/nologin ppanel
sudo install -d -o ppanel -g ppanel /var/lib/ppanel
sudo systemctl daemon-reload
sudo systemctl enable --now ppanel
sudo journalctl -u ppanel -f
```
