# Deploy the server with Docker

This guide builds an image from the current server source instead of relying
on stale or unverified prebuilt images. Docker Compose v2 is required.

```bash
git clone https://github.com/perfect-panel/server.git ppanel-server
cd ppanel-server
mkdir -p etc
: > etc/ppanel.yaml
```

Create `compose.yaml`:

```yaml
services:
  ppanel:
    build:
      context: .
    container_name: ppanel-server
    restart: unless-stopped
    ports:
      - "8080:8080"
    environment:
      PPANEL_DB: postgres://ppanel:change-me@db:5432/ppanel?sslmode=disable&TimeZone=Asia%2FShanghai
      PPANEL_REDIS: redis://:change-me@redis:6379/0
    volumes:
      - ./etc:/app/etc
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_healthy

  db:
    image: postgres:16-alpine
    restart: unless-stopped
    environment:
      POSTGRES_DB: ppanel
      POSTGRES_USER: ppanel
      POSTGRES_PASSWORD: change-me
    volumes:
      - postgres-data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ppanel -d ppanel"]
      interval: 10s
      timeout: 5s
      retries: 10

  redis:
    image: redis:7-alpine
    restart: unless-stopped
    command: ["redis-server", "--requirepass", "change-me"]
    volumes:
      - redis-data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "-a", "change-me", "ping"]
      interval: 10s
      timeout: 5s
      retries: 10

volumes:
  postgres-data:
  redis-data:
```

Replace every example password before the first start, then run:

```bash
docker compose up -d --build
docker compose logs -f ppanel
```

The first successful start writes the base configuration to
`./etc/ppanel.yaml`. Edit `JwtAuth.AccessSecret`, `Administrator`, and
`Database`, then restart the service:

```bash
docker compose restart ppanel
```

The backend listens on `http://server-address:8080` by default. Terminate
public HTTPS traffic at a reverse proxy and do not expose PostgreSQL or Redis.
MySQL/MariaDB remains supported, but PostgreSQL is recommended for new
deployments.
