# 快速开始：Docker

推荐先部署服务端，再独立构建和部署管理端与用户端。旧版“一键三镜像”示例依赖
已过期的前端镜像和配置字段，已不再推荐。

## 1. 启动服务端

```bash
git clone https://github.com/perfect-panel/server.git ppanel-server
cd ppanel-server
mkdir -p etc
: > etc/ppanel.yaml
```

使用下方最小 Compose 文件启动服务端、推荐的 PostgreSQL 和 Redis。请先替换所有
示例密码。

```yaml
services:
  ppanel:
    build: .
    restart: unless-stopped
    ports: ["8080:8080"]
    environment:
      PPANEL_DB: postgres://ppanel:change-me@db:5432/ppanel?sslmode=disable&TimeZone=Asia%2FShanghai
      PPANEL_REDIS: redis://:change-me@redis:6379/0
    volumes: ["./etc:/app/etc"]
    depends_on: [db, redis]
  db:
    image: postgres:16-alpine
    restart: unless-stopped
    environment:
      POSTGRES_DB: ppanel
      POSTGRES_USER: ppanel
      POSTGRES_PASSWORD: change-me
    volumes: ["postgres-data:/var/lib/postgresql/data"]
  redis:
    image: redis:7-alpine
    restart: unless-stopped
    command: ["redis-server", "--requirepass", "change-me"]
    volumes: ["redis-data:/data"]
volumes:
  postgres-data:
  redis-data:
```

```bash
docker compose up -d --build
docker compose logs -f ppanel
```

首次启动后，编辑 `etc/ppanel.yaml` 中的 `JwtAuth.AccessSecret` 与管理员密码，
然后执行 `docker compose restart ppanel`。生产环境请使用[完整 Docker
指南](../server/docker)，其中包含健康检查和安全说明。

## 2. 部署前端

前端仓库现在是 [perfect-panel/frontend](https://github.com/perfect-panel/frontend)，
包含 Vite 构建的 `apps/admin` 和 `apps/user`，而不是旧版 `ppanel-web`。

在两个应用各自的 `.env.production` 中设置：

```dotenv
VITE_API_BASE_URL=https://api.example.com
```

按[前端分离部署指南](https://github.com/perfect-panel/frontend/blob/main/docs/zh/guide/separation/frontend.md)
构建静态文件并通过 Nginx、对象存储/CDN 或 Vercel 部署。请确保后端 CORS 与前端
域名配置一致。
