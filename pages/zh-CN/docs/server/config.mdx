# 服务端配置

PPanel 服务端默认读取 `etc/ppanel.yaml`；可通过 `--config` 指定其他路径：

```bash
ppanel run --config /etc/ppanel/ppanel.yaml
```

首次启动时，如果默认配置文件为空，服务会读取 `PPANEL_DB` 与
`PPANEL_REDIS`，生成必要的基础配置并写回该文件。生产环境应保存生成后的
配置文件，并为 JWT 密钥设置稳定值。

## 最小配置

```yaml
Model: prod
Host: 0.0.0.0
Port: 8080
AppLocation: Asia/Shanghai

JwtAuth:
  AccessSecret: 请替换为高强度随机字符串
  AccessExpire: 604800

Database:
  Driver: postgres
  Addr: 127.0.0.1:5432
  Dbname: ppanel
  Username: ppanel
  Password: 请替换为数据库密码
  Config: sslmode=disable&TimeZone=Asia/Shanghai
  MaxIdleConns: 10
  MaxOpenConns: 10
  SlowThreshold: 1000

Redis:
  Host: 127.0.0.1:6379
  Pass: ""
  DB: 0

Administrator:
  Email: admin@example.com
  Password: 请替换为高强度初始密码
```

`Database` 是当前配置项，支持 `postgres` 与 `mysql`。推荐新部署使用
PostgreSQL；旧版 `MySQL` 配置仍兼容，但已弃用，不应在新部署中继续使用。

## 首次启动环境变量

仅在默认配置文件为空时使用下列变量完成初始化；初始化后应以 YAML 配置文件
为准。

| 环境变量 | 示例 |
| --- | --- |
| `PPANEL_DB` | `postgres://ppanel:password@127.0.0.1:5432/ppanel?sslmode=disable&TimeZone=Asia%2FShanghai` |
| `PPANEL_REDIS` | `redis://:password@127.0.0.1:6379/0` |

MySQL/MariaDB 仍受支持；如果使用它，可使用驱动 DSN：

```text
ppanel:password@tcp(127.0.0.1:3306)/ppanel?charset=utf8mb4&parseTime=true&loc=Asia%2FShanghai
```

## 运行要求

- Go 版本仅在从源码构建时需要；运行预编译二进制或容器时不需要。
- 推荐 PostgreSQL；MySQL/MariaDB 仍受支持。Redis 用于缓存与队列。
- 将数据库、Redis 和服务端口限制在可信网络中，公网入口应通过 HTTPS 反向代理。
