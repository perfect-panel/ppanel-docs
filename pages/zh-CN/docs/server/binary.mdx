# 从源码部署服务端

当需要自行构建二进制时，使用当前的
[服务端仓库](https://github.com/perfect-panel/server)。不要使用旧文档中的固定版本
下载地址或已迁移的仓库名。

## 准备依赖

- Go 1.25 或更高版本
- PostgreSQL（推荐；MySQL/MariaDB 仍受支持）
- Redis

创建 PostgreSQL 数据库和低权限账号：

```sql
CREATE USER ppanel WITH PASSWORD 'replace-this-password';
CREATE DATABASE ppanel OWNER ppanel;
```

## 构建与安装

```bash
git clone https://github.com/perfect-panel/server.git
cd server
go build -trimpath -o ppanel .

sudo install -Dm755 ppanel /usr/local/bin/ppanel
sudo install -d -m 0750 /etc/ppanel
sudo install -m 0640 etc/ppanel.yaml /etc/ppanel/ppanel.yaml
```

编辑 `/etc/ppanel/ppanel.yaml`，使用 `Database`（而不是已弃用的 `MySQL`）配置
PostgreSQL；完整示例见[服务端配置](./config)。MySQL/MariaDB 的迁移和连接方式仍
受支持，但新部署推荐 PostgreSQL。

## systemd

创建 `/etc/systemd/system/ppanel.service`：

```ini
[Unit]
Description=PPanel server
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=ppanel
Group=ppanel
WorkingDirectory=/var/lib/ppanel
ExecStart=/usr/local/bin/ppanel run --config /etc/ppanel/ppanel.yaml
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

创建运行用户与目录后启动：

```bash
sudo useradd --system --home /var/lib/ppanel --shell /usr/sbin/nologin ppanel
sudo install -d -o ppanel -g ppanel /var/lib/ppanel
sudo systemctl daemon-reload
sudo systemctl enable --now ppanel
sudo journalctl -u ppanel -f
```
