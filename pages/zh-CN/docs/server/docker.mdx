# 使用 Docker 部署服务端

本指南从当前服务端源码构建镜像，避免依赖过期或来源不明的预构建镜像。需要
Docker Compose v2。

```bash
git clone https://github.com/perfect-panel/server.git ppanel-server
cd ppanel-server
mkdir -p etc
: > etc/ppanel.yaml
```

创建 `compose.yaml`：

```yaml
services:
  ppanel:
    build:
      context: .
    container_name: ppanel-server
    restart: unless-stopped
    ports:
      - "8080:8080"
    environment:
      PPANEL_DB: postgres://ppanel:change-me@db:5432/ppanel?sslmode=disable&TimeZone=Asia%2FShanghai
      PPANEL_REDIS: redis://:change-me@redis:6379/0
    volumes:
      - ./etc:/app/etc
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_healthy

  db:
    image: postgres:16-alpine
    restart: unless-stopped
    environment:
      POSTGRES_DB: ppanel
      POSTGRES_USER: ppanel
      POSTGRES_PASSWORD: change-me
    volumes:
      - postgres-data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ppanel -d ppanel"]
      interval: 10s
      timeout: 5s
      retries: 10

  redis:
    image: redis:7-alpine
    restart: unless-stopped
    command: ["redis-server", "--requirepass", "change-me"]
    volumes:
      - redis-data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "-a", "change-me", "ping"]
      interval: 10s
      timeout: 5s
      retries: 10

volumes:
  postgres-data:
  redis-data:
```

在首次启动前替换示例中的所有密码，然后执行：

```bash
docker compose up -d --build
docker compose logs -f ppanel
```

首次成功启动会将基础配置写入 `./etc/ppanel.yaml`。随后编辑该文件中的
`JwtAuth.AccessSecret`、`Administrator` 与 `Database` 等项目，并重启服务：

```bash
docker compose restart ppanel
```

服务端默认监听 `http://服务器地址:8080`。生产环境请在反向代理处配置 HTTPS，
不要直接公开 PostgreSQL 或 Redis 端口。MySQL/MariaDB 仍可使用，但推荐新部署使用
PostgreSQL。
